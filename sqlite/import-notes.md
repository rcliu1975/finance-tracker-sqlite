# 匯入來源備註

這份文件記錄目前已確認的匯入來源格式，避免之後寫 SQLite 匯入腳本時重新猜測。

## `~/item_2025_UTF-8_import.csv`

已確認欄位：

- `類別`
- `項目名稱`
- `期初餘額`
- `次序`
- `常用摘要`

範例：

```text
資產,現金,314270,0,
```

說明：

- 這和現有前端 `匯入項目` 流程的預期欄位一致。
- `常用摘要` 可直接對應到 SQLite 規劃中的 `common_summaries`。

## `~/transactionP-2025_utf-8_import.csv`

已確認欄位：

- 空白欄位（看起來像原始序號欄）
- `日期`
- `從項目`
- `至項目`
- `金額`
- `摘要`
- `備註`

範例：

```text
1,2009/8/8,國泰世華costco卡,雜物費,48888,電視,電視 $26888 / 洗衣機 $22000
```

說明：

- 第一欄目前可視為來源檔序號，SQLite 匯入時可忽略，不要寫進核心交易表。
- `日期` 使用 `YYYY/M/D`，匯入時需要正規化成 `YYYY-MM-DD`。
- `備註` 可能含換行，匯入腳本要以真正的 CSV parser 處理，不能用單純 `split(\"\\n\")`。
- 現有前端匯入欄位假設與這份交易檔大致相容，但實作匯入腳本時要顯式處理前導空白欄位。
